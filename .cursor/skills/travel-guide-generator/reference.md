# 旅游攻略生成器 - 参考说明

## 产品改进说明（修改建议与落地）

以下为产品视角的修改建议及在 skill 中的落地方式，便于用户更易用、搜索更丰富精准、MD 更易读、HTML 更丰富。

| 方向 | 建议 | 落地 |
|------|------|------|
| **易用** | 快捷入口、少轮对话 | 支持「先只做到 MD」「直接生成 HTML」等分步执行；提供信息预填模板，一次填空算完成度 |
| **易用** | 阶段可预期 | 阶段切换时明确话术：「信息已够，开始查资料」「下面输出 MD 供确认」 |
| **搜索** | 渠道更丰富 | reference 中按类型列渠道：综合攻略、餐厅、住宿、交通、门票、UGC、当地安全等，含携程/飞猪/GetYourGuide/Reddit/知乎/抖音/B站等 |
| **搜索** | 内容更精准 | 按信息类型给检索式（开放时间、人均、避坑、近期体验等）；重要信息两处来源交叉验证；注明来源与时效 |
| **MD** | 更易读 | 增加目录（锚点到每日）、统一图标（⏰📍💰⚠️📸🔗）、可选一页速览表（日期\|早\|上午\|午\|下午\|晚\|备注） |
| **HTML** | 更丰富 | 首页元信息卡片（目的地/日期/天数/预算/标签）；可选模块：交通时刻表、预算汇总、紧急联系、图片位；移动端点击区域与可访问性 |

## 一、信息收集必采清单（与完成度对应）

用于多轮对话时逐项确认，未填项在「仍缺」中列出。

| 序号 | 字段 | 权重 | 示例/说明 |
|------|------|------|----------|
| 1 | 目的地（国家/城市/地区） | 15% | 如：日本东京+长野、意大利罗马+佛罗伦萨、中国云南 |
| 2 | 出行日期与天数 | 15% | 如：2026年3月13日–17日、共5天 |
| 3 | 出行季节/月份 | 5% | 如：3月春季、2月冬季，用于配色与穿衣 |
| 4 | 偏好类型 | 10% | 美食/人文/自然/购物/亲子/滑雪/海岛等，可多选 |
| 5 | 预算档次 | 5% | 经济/舒适/轻奢/不设限 |
| 6 | 交通方式与节点 | 10% | 航班、火车、自驾、机场/车站名、是否夜巴等 |
| 7 | 住宿区域或已定酒店 | 5% | 如：罗马中央火车站附近、白马高山酒店 |
| 8 | 必去/必吃清单 | 10% | 景点、餐厅、活动名称 |
| 9 | 禁忌与限制 | 5% | 不吃辣、不去人多、身体限制等 |
| 10 | 特殊需求 | 10% | 签证、退税、防盗、预约、药妆、紧急电话等 |
| 11 | 地图偏好 | 5% | 默认：中国境内→高德，境外→Google |
| 12 | 其他备注 | 5% | 自由补充 |

完成度 = 已明确填写的字段对应权重之和，目标 ≥85% 再进入资料搜集与写作。

## 四、地图链接规则

### 中国境内（高德地图）

- 搜索 URL：`https://uri.amap.com/search?keyword=关键词` 或 `https://www.amap.com/search?query=关键词`
- MD 中：`[地点名](https://uri.amap.com/search?keyword=地点名)`
- HTML 中：`<a href="https://uri.amap.com/search?keyword=地点名" target="_blank" rel="noopener" class="loc">地点名</a>`

### 中国境外（Google 地图）

- 搜索 URL：`https://www.google.com/maps/search/?api=1&query=关键词`
- MD 中：可写「**地点名**」或 `[地点名](https://www.google.com/maps/search/?api=1&query=地点名)`
- HTML 中：使用点击打开新窗口，与现有项目一致：

```html
<span class="loc" onclick="openMap('地点名')">地点名</span>
```

```html
<script>
function openMap(loc){
  window.open('https://www.google.com/maps/search/?api=1&query='+encodeURIComponent(loc),'_blank');
}
</script>
```

若整站均为中国境内，可全部采用高德并统一用 `<a>`；若为境外或混合，境外地点用 `openMap`，境内地点用高德 `<a>`。

## 七、精准检索式建议

按「信息类型」组织关键词，提高命中率与时效性。

| 信息类型 | 检索式示例 | 说明 |
|----------|------------|------|
| 开放时间/门票 | `[景点名] 开放时间 2025`、`[景点名] 门票 预约` | 带年份减少过期信息 |
| 排队/预约 | `[景点名] 排队 预约`、`[目的地] 免排队 攻略` | 用于写「重要提示」 |
| 餐厅人均/招牌 | `[餐厅名] 人均`、`[目的地] 必吃 餐厅` | 人均写进最优项 |
| 避坑/注意 | `[目的地] 避坑`、`[目的地] 注意 防盗`、`[目的地] 踩雷` | 写入提示与时间段注意 |
| 小众/拍照 | `[目的地] 小众`、`[景点名] 拍照 机位` | 备选或拍摄点 |
| 交通时刻 | `[起点] 到 [终点] 火车/大巴`、`[机场] 到 [市区] 交通` | 写入当日交通块 |
| 近期体验 | `[目的地] 2025 游记`、`[目的地] 最近 攻略` | 优先近期 UGC |

重要结论（如门票价格、预约链接、开放时间）尽量**两处来源一致**再写进攻略；若不一致，可写「约 xxx」「建议出发前在官网确认」。

## 八、MD 易读规范（目录/图标/速览）

- **目录**：在标题与副标题下、行程概览前，增加「## 目录」，用 Markdown 锚点链接到各「## 🗓️ 月/日」。
- **统一图标**：⏰ 时间段、📍 位置、💰 人均/预算、⚠️ 注意、📸 拍摄点、🔗 链接；全篇统一，便于扫读。
- **一页速览表**（可选）：在行程概览表后增加「## 一页速览」，表格示例：

| 日期 | 早餐 | 上午 | 午餐 | 下午 | 晚餐 | 备注 |
|------|------|------|------|------|------|------|
| 3/13 | 酒店/机场 | 人民广场→西班牙阶梯 | Ciampini Gelato | 许愿池 | Hostaria Romana 四大面 | 防盗注意 |
| … | … | … | … | … | … | … |

每格一句话，方便用户快速把握整天安排后再看详细时间段。

## 九、HTML 丰富度与可选模块

在保证与现有 snow/anti-theft 结构一致的前提下，可增加以下内容使页面更丰富：

| 模块 | 位置 | 说明 |
|------|------|------|
| **元信息卡片** | 首页副标题下、概览卡片前 | 展示：目的地、出行日期、天数、建议人数/预算区间、标签（美食/人文/自然等）；用 `<table>` 或标签列表，样式与 `.tips` 区分 |
| **交通时刻小表** | 涉及火车/大巴的当日区块内 | 用 `.info-table` 列出车次、发车时间、站点、备注（预约链接等） |
| **预算/花费汇总** | 首页或单独一节 | 可选：总预算区间、按日或按类（交通/住宿/餐饮/门票）粗估；用表格或列表 |
| **紧急联系** | 首页 tips 内或页底 | 报警、使领馆、急救、旅游投诉等；与重要提示统一风格 |
| **图片** | 最优项内或当日头图 | `img/` 下图片，`loading="lazy"`，`alt` 写清说明；无图可占位 `<div class="spot-img placeholder">图位</div>` 或省略 |
| **移动端友好** | 全局 | 按钮与 `.loc` 最小点击区域约 44px；导航 `flex-wrap`；小屏下表格横向滚动或简化为列表 |

可访问性：标题层级连续（h1 → h2 → h3）；关键交互元素可聚焦；色差足够（主色与背景对比度达标）。
